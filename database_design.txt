DATABASE DESIGN (ALIGNED TO database_classes.py)

GENERAL RULES
- Every object or process step stores an immutable `entry_created_date` (`YYYYMMDD_HHMMSS`).
- Every object or process step stores an immutable unique ID.
- `Sample` IDs: 6 hex characters.
- `Processing_Step` IDs: `P` + 5 hex characters.
- Properties are stored as dictionaries (`obj.properties`).
- Optional universal property: `date` (if provided by user).
- Keys are tracked in `database_keys.txt` as `ClassName_property`.
- Unknown keys are allowed but trigger a warning.

TREE STRUCTURE
- Root: SYSTEM node with data key `Sample_System`.
- Logical structure: sample system > object > sub-object > ... > process step.
- Parent-child constraints are controlled by each class `permitted_children` list.

BASE CLASSES

1) Sample
- Purpose: physical sample/object node.
- Required properties: provided by subclasses.
- Common behavior:
	- required property validation
	- immutable ID and creation date
	- optional `date`
	- key logging to `database_keys.txt`

2) Processing_Step (inherits Sample)
- Purpose: process/action node.
- Required properties: provided by process-step subclasses.
- Common behavior:
	- required property validation
	- immutable ID and creation date
	- key logging to `database_keys.txt`
	- default `permitted_children = []`

NOT YET IMPLEMENTED VS. ORIGINAL CONCEPT NOTES
- `TEM_grid` is not implemented as a class. Current closest class: `TEM_lamella`.
- `TEM imaging` and `micropillar testing` exist as generalized classes:
	- `Imaging` (microscope required)
	- `Micromechanical_testing` (`test_type` required)
- `sample system` is represented as a root SYSTEM node, not as a Python class.

GUI-RELEVANT DESIGN NOTES (CURRENT IMPLEMENTATION)
- Child class options are dynamically filtered from `permitted_children` of the selected parent.
- Required properties are loaded from `required_properties.txt`.
- Tree persistence is JSON-based (`databases/*.json`).
- Existing object IDs and creation dates are restored during deserialization.

IMPLEMENTED GUI FEATURES
- Sorting: Nodes can be sorted by Name, Date Created, or None (preserves tree order). Sorting works in both classic and rainbow modes.
- Node display enhancements:
	- For materials containing "calibration", displays `sputter_current` and `sputter_flow` values after material name (e.g., "Ag calibration (50, 10 sccm)").
	- For `XRay_analysis` nodes, displays the `mode` property after node name (e.g., "XRay_analysis (XRR)").
	- For `Annealing` nodes, displays `temperature_C` (e.g., "Annealing (500°C)").
	- For `Micromechanical_testing` nodes, displays `test_type`.
- Rainbow mode: Color-codes tree depth; respects active sort mode.
- Multi-tree view: Support for loading and viewing multiple trees simultaneously with individual sort state.
- Side panel: "Node Properties" panel displays properties of the currently selected node.
- Copy node: "Copy Node" button allows cloning a node (with class and properties) to another parent.
- Search functionality: Global search with regex, substring, or exact matching across all trees; supports filtering by node type and property key/value pairs.
